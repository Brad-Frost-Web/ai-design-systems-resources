A pull request is a fundamental part of collaborating on code: after you push your work up (for example, purple buttons), you issue a pull request that essentially says "Hey, I made the buttons purple — what do you think?" proposing to merge a feature branch into another branch like main. It pulls up a diff of the changes and lets you tag people to review the code and have a discussion before it's merged. The course notes teams can set up workflows requiring reviews and checks to ensure quality before merging — really important in the AI-powered age.
