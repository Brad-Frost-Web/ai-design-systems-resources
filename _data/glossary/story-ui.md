A tool created by TJ that lets you interface with LLMs using the actual context of your design system, rather than approximating it or assuming React with Tailwind. It points at your components directory and knows your design tokens, so it is framework-agnostic (React, Vue, Angular, Svelte, web components) and model-agnostic (Claude, OpenAI, Gemini). Installed via npm into the package where Storybook lives and configured through a setup wizard, it adds an in-Storybook generator that creates new compositions, pages, and recipes from the real components. It also validates generated code, can latch onto the Storybook MCP, and offers a voice canvas so cross-disciplinary stakeholders can speak to the LLM and iterate on designs together in real time.
