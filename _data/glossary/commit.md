Committing code means capturing the edits or updates you have made (for example, changing the buttons to purple) along with a message describing the change. Each commit is a snapshot in the project's history, and stepping through that history lets you undo, roll back, and revert to previous versions. The course notes that committed work is captured by Git so there is no outstanding untracked work, and that AI is genuinely helpful for writing good, descriptive commit messages instead of vague ones like "work in progress."
