The course's term for using AI and MCP tools to convert assets between design and code in either direction. Design-to-code translates an AI-ready Figma component into a working coded component (markup, styles, stories, tests) that respects the existing codebase's conventions and design tokens; code-to-design pushes something that exists in code (such as an inverted variant) back into the Figma file. The course frames this two-way movement as a 'swirly cloud' where it no longer matters whether you start from design or code, as long as you have tooling to keep both sides synchronized.
