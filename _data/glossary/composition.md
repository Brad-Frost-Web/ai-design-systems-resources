In Story UI, a higher-order assembly (a page, pattern, or recipe) built by combining existing design system components, as opposed to building individual component APIs. The course frames design systems as not just about building components but about actually using and stress-testing them in real products. Story UI generates a composition from a natural-language prompt (e.g., a sign-in page with username field, password field, save-password checkbox, and buttons) using the real components, tokens, card, logo, and alert from the design system, letting stakeholders iterate together.
