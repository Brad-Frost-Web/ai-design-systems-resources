The main branch is the primary, stable line of code in a repository — typically the live website or the published library. The whole point of branching is to keep main stable while you experiment elsewhere; feature work is only merged back into main once it is committed, reviewed, and approved. The course shows main as the branch that ultimately powers the real website.
