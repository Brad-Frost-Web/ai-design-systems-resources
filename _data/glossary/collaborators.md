Collaborators are the people you invite to a GitHub repository and grant permission to work on the project, such as write access to push code. The course shows adding collaborators (Ian, TJ) via Settings > Collaborators, and frames this access management — deciding who can write code where — as a core thing platforms like GitHub and GitLab give you, while noting that not just any random account should be able to write to a company's codebases.
